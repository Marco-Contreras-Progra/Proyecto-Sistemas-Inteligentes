Comprender el agente de aprendizaje por refuerzo

Desde el estado hasta la recompensa

Este módulo explica cómo el agente interpreta una transacción, toma una decisión y ajusta su tabla de valores para mejorar con el tiempo. La idea central es que cada caso deja una señal útil para futuras evaluaciones.

1. Observa

Revisa las señales del caso: monto, ubicación, dispositivo, hora y frecuencia.

2. Decide

Escoge entre aprobar o bloquear según la política aprendida.

3. Aprende

Recibe una recompensa y ajusta la valoración de esa acción para el futuro.

Fundamentos de RL

El agente observa cada transacción como si fuera un inspector: recibe una compra, revisa sus señales y decide si aprobarla o bloquearla. Su objetivo no es adivinar al azar, sino aprender de cada resultado.

El estado está formado por 5 señales de la compra: monto, ubicación, dispositivo, hora y frecuencia. Cada combinación se convierte en una situación distinta para el agente.

El sistema usa una versión sencilla de Q-Learning. Cada vez que el agente actúa, compara lo que esperaba con lo que realmente pasó y ajusta su memoria:

\(Q(s,a) \leftarrow Q(s,a) + \alpha\,[R + \gamma\,\max_{a'} Q(s',a') - Q(s,a)]\)

La idea es simple: si una decisión trae buen resultado, se refuerza; si trae problemas, se debilita para futuras ocasiones.

α = 0.2 · Tasa de aprendizaje

Hace que el agente aprenda de forma gradual, sin corregir demasiado rápido.

γ = 0.0 · Factor de descuento

Al ser un problema de Bandido Contextual (las transacciones son independientes entre sí), se configura en 0 para enfocarse 100% en la recompensa inmediata.

ε = 0.15 · Exploración

Permite probar decisiones nuevas en algunos casos, para descubrir mejores reglas.

Conclusiones y Limitaciones

El Aprendizaje por Refuerzo demuestra ser una herramienta poderosa para la detección de anomalías en transacciones. Al interactuar continuamente con el entorno, el agente logra adaptarse a patrones de fraude sin depender exclusivamente de reglas estáticas.

Limitaciones del modelo propuesto:

  • Fase de exploración inicial: Al inicio (cuando la tabla Q está vacía), el agente cometerá errores costosos (falsos positivos o negativos) mientras explora.
  • Entorno Adversario: En el mundo real, los estafadores cambian sus tácticas constantemente. El modelo necesita actualización continua para no volverse obsoleto.
  • Bandido Contextual vs RL Completo: Dado que las transacciones son eventos independientes, este modelo se simplificó usando un factor de descuento (γ = 0), limitando la capacidad de predecir cadenas de eventos a largo plazo.
